# GradeMaster: System Overview

## 📋 Methodology
GradeMaster utilizes a **Z-Score Normalization** model to ensure objective grading across diverse evaluation components. By calculating the standard deviation ($\sigma$) and mean ($\mu$) of the class, the system measures each student's relative performance, neutralizing potential evaluation bias.

## 📊 Data Pipeline
1. **Template Definition**: User defines evaluation components (e.g., Mid Sem, Group Project) with specific weights and max marks.
2. **Data Integration**:
   - **Student Data**: Individual scores for non-team components.
   - **Team Mapping**: A relational file linking Student IDs to Team IDs.
   - **Team Marks**: Collective scores for group-based tasks.
3. **Validation**: The system cross-references all three inputs to ensure 100% data consistency.
4. **Scoring Engine**:
   - Computes Weighted Totals (standardized to 100%).
   - Calculates absolute point totals.
   - Applies outlier strategies (Cap/Remove/Ignore).
5. **Interactive Grading**: Faculty adjust boundaries on a dynamic Histogram, which triggers automatic grade re-assignment via an optimized **MaxHeap** structure.

## 🔍 Outlier Handling
- **Statistical Capping**: Instead of discarding extreme performance data, the system limits outliers to $2\sigma$ from the mean. This preserves the sample size while stabilizing the standard deviation.

## 📥 Exports
Exports are generated in `.xlsx` format, featuring:
- Rank-ordered results.
- Statistical Metadata (Mean, Std Dev, Z-Scores).
- Final Grade assignments.
- Toggle-ready score columns (Weighted vs Raw).
